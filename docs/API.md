# ReactoN Advanced Mathematical & API Reference

This document serves as the official scientific and API reference for the **ReactoN** computational engine. It details the mathematical models, chemical kinetics, thermodynamic balances, and structural mechanics equations implemented in the codebase.

---

## 🔬 Core Scientific Models

### 1. Chemical Kinetics: Shrinking Core Model (SCM)
For a solid-liquid reaction involving spherical aluminum particles of Sauter mean diameter ($d_{32} = 2 R_0$), the reaction progress (conversion fraction $\alpha \in [0, 1]$) is governed by two limiting regimes:

#### A. Chemical Reaction Controlled Regime:
When diffusion through the liquid film and porous ash layer is fast, the rate is controlled by the chemical reaction at the core boundary:
$$t = \tau_{rxn} \left[ 1 - (1-\alpha)^{1/3} \right]$$
where:
$$\tau_{rxn} = \frac{\rho_m R_0}{k_{rxn} C_b}$$
-   $\rho_m$: Molar density of aluminum ($\text{mol/m}^3$)
-   $R_0$: Initial particle radius ($\text{m}$)
-   $C_b$: Bulk concentration of activating hydroxyl ions ($[OH^-]$, $\text{M}$)
-   $k_{rxn}$: Temperature-dependent Arrhenius rate constant:
    $$k_{rxn}(T, \text{pH}) = A \cdot \exp\left(-\frac{E_a}{R T}\right) \cdot \left[OH^-\right]^n$$

#### B. Diffusion through Porous Ash Layer Controlled Regime:
When the reaction product (aluminum hydroxide $\text{Al(OH)}_3$ and oxides) accumulates around the shrinking core, water diffusion through this porous ash layer controls the rate:
$$t = \tau_{diff} \left[ 1 - 3(1-\alpha)^{2/3} + 2(1-\alpha) \right]$$
where:
$$\tau_{diff} = \frac{\rho_m R_0^2}{6 D_e C_b}$$
-   $D_e$: Effective diffusion coefficient in the porous ash layer ($\text{m}^2/\text{s}$), modeled with Arrhenius activation:
    $$D_e(T) = D_{e,0} \cdot \exp\left(-\frac{E_{a,diff}}{R T}\right)$$

### 2. Phase Equilibrium: Dalton & Antoine Equations
The raw gas generated by the reactor is a mixture of hydrogen ($\text{H}_2$) and saturated water vapor ($\text{H}_2\text{O}$).

#### A. Saturation Vapor Pressure ($P_{sat}$):
Calculated using the **Antoine Equation**:
$$\log_{10} P_{sat} (\text{bar}) = A - \frac{B}{T(^\circ\text{C}) + C}$$
*(For water: $A = 5.11564$, $B = 1687.537$, $C = 230.17$)*

#### B. Saturated Gas Composition:
By Dalton's Law of Partial Pressures, the total pressure is:
$$P_{total} = P_{H2} + P_{H2O}(T)$$
The maximum dry hydrogen purity (mole fraction) at the condenser exit is:
$$y_{H2} = \frac{P_{total} - P_{sat}(T_{cond})}{P_{total}}$$

### 3. Mechanical Sizing: ASME BPVC Section VIII Division 1
To withstand high internal pressures, reactor vessels are sized according to ASME design stress limits:

#### A. Cylindrical Shell Wall Thickness ($t_s$):
$$t_s = \frac{P \cdot R}{S \cdot E - 0.6 P} + C_a$$

#### B. 2:1 Ellipsoidal Head Wall Thickness ($t_h$):
$$t_h = \frac{P \cdot D}{2 S \cdot E - 0.2 P} + C_a$$
-   $P$: Design pressure (MPa)
-   $R$: Inside shell radius (mm)
-   $D$: Inside head diameter ($2R$, mm)
-   $S$: Allowable material stress (e.g. 138 MPa for 316SS)
-   $E$: Joint efficiency weld factor (e.g. 0.85)
-   $C_a$: Corrosion allowance (mm)

## 📡 REST API Route Specifications

The FastAPI server exposes these mathematical engines via three primary POST endpoints:

### 1. `POST /api/v1/simulate`
Simulates a batch reactor run over a given timeline.

-   **Input JSON Schema (`SimulationInput`):**
    ```json
    {
      "aluminum_mass_kg": 0.5,
      "water_mass_kg": 2.5,
      "d_ox_nm": 3.0,
      "particle_size_mean_um": 25.0,
      "particle_size_std_um": 5.0,
      "pH": 12.5,
      "water_type": "tap",
      "catalyst_molar": 0.1,
      "reactor_volume_l": 5.0,
      "cooling_duty_w": 200.0,
      "initial_temp_c": 25.0,
      "initial_pressure_bar": 1.0,
      "dt": 1.0,
      "max_time_s": 600.0
    }
    ```
-   **Response JSON Schema (`SimulationOutput`):**
    Returns lists of high-frequency telemetry rows containing `temperature_c`, `pressure_bar`, `conversion_fraction`, `H2_produced_g`, and `in_induction` states.

---

### 2. `POST /api/v1/optimize`
Solves multivariable operating parameters to maximize yield or minimize LCOH.

-   **Input JSON Schema (`OptimizationInput`):**
    ```json
    {
      "objective": "maximize_h2_rate",
      "feedstock_type": "premium",
      "water_type": "tap",
      "target_purity": 0.999,
      "max_pressure_bar": 16.0,
      "max_temp_c": 95.0
    }
    ```
-   **Response JSON Schema (`OptimizationOutput`):**
    Returns `optimal_parameters` (feed rate, water ratio, pressure, temperature) and resulting `metrics` (SLPM, purity, LCOH, exergy efficiency).

---

### 3. `POST /api/v1/compliance/asme`
Calculates ASME wall thicknesses.

-   **Input JSON Schema (`ASMECalculationInput`):**
    ```json
    {
      "design_pressure_bar": 12.0,
      "reactor_radius_mm": 120.0,
      "allowable_stress_mpa": 138.0,
      "joint_efficiency": 0.85,
      "corrosion_allowance_mm": 1.5
    }
    ```
-   **Response JSON Schema (`ASMECalculationOutput`):**
    Returns calculated shell and head thicknesses in millimeters.
